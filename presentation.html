03 / 11

💡 核心洞察

从访谈中提炼的 4 条洞察句（Insight Statements）：

#1 选择疲劳

当都市白领在几十个品牌/SKU中挑选时，想要快速找到"属于自己的口味"，但是现有品牌只提供固定产品线，无法匹配个体偏好。

#2 反馈缺失 ⭐

当用户尝试咖啡订阅时，想要每次都是惊喜但不踩雷，但是现有订阅缺乏反馈闭环，推荐与偏好脱节。

#3 仪式感丧失

当咖啡变成"功能性提神工具"时，想要找回"为自己挑选一杯好咖啡"的仪式感，但是便利化让所有选项趋同。

#4 信任门槛

当用户愿意为个性化付溢价时，想要看到 AI 确实比自己选得好，但是对"AI"标签存在"噱头感"的戒心。

▶ 选择洞察 #2 进入 HMW——因为它直指可工程化解决的产品机制缺失

04 / 11

🤔 How Might We 发散

基于核心洞察 #2，我们提出 5 个 HMW 问题：

HMW #1 ⭐

如何让用户的每一次口味反馈都"看得见"地改善下一次推荐？

HMW #2

如何在用户还没明确表达偏好时，就从行为中学到他们的口味？

HMW #3

如何让"探索新口味"变成零风险的愉悦体验而非赌博？

HMW #4

如何用最少的操作成本完成一次有效的口味反馈？

HMW #5

如何让推荐系统的"成长过程"本身成为用户留存的理由？

▶ 选择 HMW #1：可见的反馈闭环 — 解决信任问题的关键

05 / 11

🧠 点子生成

围绕 HMW #1，团队生成了 3 个方案概念：

A · 味觉日记

每杯喝完做10秒评价（滑块/emoji），生成可视化「味觉雷达图」并实时更新，下期配方基于雷达图调整。

关键词：可视化成长

B · 盲盒进化

每期寄 3 小包（A/B/C），用户只需选"最喜欢哪个"，系统用对比学习快速收敛偏好，无需描述口味。

关键词：低认知负担

C · 咖啡 Spotify

仿 Discover Weekly：每期 80% 安全区 + 20% 探索区，配合每月「味觉年报」制造分享时刻。

关键词：可控冒险 + 社交

🚀 产品实作 · MVP

我们构建了「调调 Brew DJ」的 Landing Page + 味觉问卷原型：

Landing Page

  • 品牌主张："你的 AI 味觉 DJ"
  • 价值承诺：越喝越懂你
  • CTA：开始你的味觉之旅
  • 社会认证：味觉人格分享卡

核心功能原型

  • 5维味觉雷达图（酸/苦/甜/醇厚/花果香）
  • 10秒快速反馈（emoji + 滑块）
  • 推荐理由卡片："因为你上次…"
  • 月度味觉年报 + 分享组件

技术架构

  • 前端：静态站（GitHub Pages）
  • 推荐：协同过滤 + LLM 味觉推理
  • 反馈链路 ≤15秒完成
  • 数据飞轮：推荐→反馈→优化→推荐

定价策略

  • 客单价：8-12元/杯
  • 月订阅：199-299元（12-20杯）
  • 溢价来源：个性化 + 品质稳定性
  • 对标：三顿半与Manner之间

🧪 实验设计

核心假设

25-35岁都市白领愿意为 AI 个性化配方的咖啡订阅付出 8-12元/杯的溢价（高于三顿半5-8元），且反馈闭环能使续订率 > 60%。

实验方法

  • Landing Page 上线 → 收集邮箱/微信
  • 味觉问卷完成率追踪
  • 模拟推荐结果满意度打分
  • 付费意愿调研（CTA 点击 → 价格页）

关键指标

  • Landing → 问卷转化率（目标 >30%）
  • 问卷完成率（目标 >70%）
  • 价格页点击率（目标 >15%）
  • 邮箱留资率（目标 >10%）

时间框架

  • 第1周：上线 Landing + 社媒投放
  • 第2-3周：收集数据 + 用户回访
  • 第4周：数据分析 + 迭代决策
  • 渠道：小红书种草 + 朋友圈裂变
